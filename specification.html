JSON Response Format and Structure


{"success":boolean,

"error":boolean,

"errorData":{"code":123,

"message":"simple error message string intended for end user",

"details":"optional detailed error string intended for developers should go to javascript

console and not the browser window",

"trace":"optional stack trace or equivalent error diagnostic report, should go to javascript

console and not the browser window"},"

dataType":text/html/number/json-object/json-array,

"data":"text or HTML"/{"json":"object"}/["json","array"],

"responseSent":{"timestamp":1348934441,

"formattedDate":"Saturday, September 9th, 2012"}

"debug":"optional debug message, should go to javascript console and not the browser"}


JSON Response primary fields:


Property Name

Required/Optional

Description

Data Types

success

Required

Indicates if the request was sucessfull

Boolean

error

Required

Indicates if an error occurred.

Boolean

errorData

Optional

Used only if an error has occurred.

JSON Object

dataType

Required

Indicates the type of data stored in the data property. Valid types are: "text:,"html","number","json-object","json-array". Although the type can be any one of these values, the data property can still be set to null.

String

data

Required

The main content of the response. Usually set to null when an error or other request failure occurs.

Text, HTML,Number, JSON Object or JSON Array.

responseSent

Required

A JSON Object containing the Unix timestamp and formatted date string when the response was sent.

JSON Object

debug

Optional

A per-request debugging message.

String
